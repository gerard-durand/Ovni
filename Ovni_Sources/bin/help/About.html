Outil de VisualisatioN Interactif

OVNI a été développé à l'Onera afin d'éditer, modifier et corriger des bases de données d'avions pour CRIRA au format SDM d'Oktal.
Il peut importer d'autres formats de base de données comme les fichiers WaveFront Obj (.obj), Autodesk (.3ds), Milkshape 3D texte (.m3d), Polygon File de Niratam (.ply), Object File Format (.off) et Groupe 3D Onera (.g3d).
Quelque soit le format importé, les fichiers seront enregistrés par défaut au format SDM (.bdd), seul format connu de Crira. L'exportation vers les formats .obj, .off et .g3d a été implémentée en 2020.

Il a d'abord été conçu sous la forme de projets tutorés par des élèves de l'IUT d'informatique d'Arles (2005-2006), lors de stages effectués à l'Onera de Salon de Provence

Versions précédentes avec Tcl/Tk

Les versions précédentes d'Ovni utilisaient Tcl/Tk pour créer et gérer l'interface graphique. Les fonctions utilisée dans Ovni mais ne préexistant pas dans Tcl/Tk ont été écrites en C et sont appelées depuis les scripts de Tcl/Tk. 

Pour cela, il était nécessaire d'utiliser des outils intermédiaires comme une librairie dynamique Togl (.dll pour Windows ou .so pour Linux) et mktclapp.
Togl permet d'introduire et gérer une fenêtre OpenGL dans Tcl/Tk.
Mktclapp assemble les scripts Tcl/Tk et le code compilé d'Ovni sous la forme d'un exécutable standard intégrant tout ce qu'il faut. Sans mktclapp, on peut utiliser le code compilé sous forme d'une librairie dynamique (Ovni.dll par exemple sous Windows), mais cela nécessite d'avoir une installation Tcl/Tk complète et fonctionnelle, ce qui n'est pas forcément le cas sur les différentes machines où on souhaite utiliser Ovni.

Ces outils n'ont pas changé depuis quelques années et, surtout, ne sont plus maintenus par leurs auteurs respectifs. Des incompatibilités sont apparues depuis les versions 8.6 de Tcl/Tk.


Version wxWidgets

Les incompatibilités avec les dernières version de Tcl/Tk et surtout l'abandon de certains outils intermédiaires ont motivé la réécriture d'Ovni en utilisant maintenant la librairie graphique wxWidgets. C'est une bibliothèque maintenue régulièrement par de nombreux auteurs, et sous licence GNU. Elle est utilisée dans de nombreux logiciels libre ou du commerce. Elle est nativement supportée et intégrée à l'environnement de développement Code::Blocks, qui existe sur la plupart des OS actuels (Windows, Linux, OSX), compilée en C++ par gcc/g++ notamment. De plus, il est possible de débuguer un exécutable via gdb, ce qui n'était pas faisable dans la version utilisant Tcl/Tk. La réécriture de la partie interface d'Ovni, s'est aussi accompagnée d'une migration de C vers C++. C'est un travail qui a été commencé par un stagiaire à l'Onera de Palaiseau, en 2015, mais non terminé faute de temps suffisant.


Ovni est actuellement compilé sous Windows par gcc 12.2 et utilise wxWidgets 3.21 (ou supérieur). On utilise de préférence une compilation générant un exécutable Windows en 64 bits. L'exécutable a été testé sous Window 7 et 10. On peut toutefois se contenter d'une génération en 32 bits, sous réserve que, sur de grosses bases de données, on risque d'avoir des soucis de gestion de mémoire.

Les versions précédentes, avec Tcl/Tk, ont aussi été générées pour tourner sous Linux. C'est toujours possible avec cette nouvelle version, mais n'a pu être testé : quelques ajustements seront sans doute à prévoir.
